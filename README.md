# Stock Prediction — AutoResearch 方式の日本株スクリーニングモデル

## コンセプト

Karpathy の [AutoResearch](https://github.com/karpathy/autoresearch) を株価予測に転用する。
Claude Code に自律実験ループを回させ、**日本株の値上がり期待銘柄を見つけるスコアリングモデル**を自動最適化する。

## 目標

| 項目 | 内容 |
|------|------|
| **対象** | 日本株（東証プライム・スタンダード） |
| **投資スタイル** | 中期（1〜3ヶ月） |
| **目的** | 値上がり期待の高い銘柄をスコアリングして上位をリストアップ |
| **出力** | 毎週「注目銘柄トップ10〜20」 |

## AutoResearch の仕組み

**3ファイル = AutoResearch の全て（合計 ≈ 630行）**

| ファイル | 役割 | 誰が触る？ | 行数目安 |
|---------|------|-----------|---------|
| `prepare.py` | データダウンロード / トークナイザー / データローダー / 評価ユーティリティ | FIXED（触らない） | ~200行 |
| `train.py` | モデル定義 / 最適化 / 訓練ループ / ハイパーパラメータ | AIが編集 | ~350行 |
| `program.md` | 実験の方向性 / 探索空間の制約 / 優先順位の指示 | 人間が記述 | ~80行 |

### 自律実験サイクル（×100回/晩）

```
program.md を読む
  ↓
train.py を修正（アーキテクチャ・HP変更）
  ↓
5分間訓練
  ↓
評価指標は改善した？
  ├─ Yes → git commit（改善を記録）
  └─ No  → git reset（変更を破棄）
  ↓
繰り返し（8時間 × 12回/時 = 約100実験）
```

### 評価指標（オリジナル）

- **val_bpb**（検証 bits per byte）— 低い = 良い
- 語彙サイズ非依存 → アーキテクチャ変更を公正に比較
- 単一GPU（H100）で動作
- 固定5分制 → モデルサイズ・バッチサイズが変わっても公正比較

## 株価予測版（AutoStock）の設計

### ファイル構成

| ファイル | 役割 |
|---------|------|
| `prepare.py` | 日本株データ取得（yfinance / J-Quants API）/ 特徴量生成 / 評価関数 |
| `train.py` | スコアリングモデル（LightGBM → 発展的にLSTM/Transformer）+ 訓練ループ |
| `program.md` | 実験ルール・禁止事項・探索方向性 |

### 評価指標

- **スコア上位20銘柄の N日後（20〜60営業日）平均リターン**
- ベンチマーク（TOPIX）との差分（超過リターン）
- Sharpe Ratio も副次指標として計測

### 特徴量の候補

| カテゴリ | 特徴量 | 意味 |
|---------|--------|------|
| モメンタム | 5日/20日/60日リターン | 短期〜中期のトレンド |
| 出来高 | 出来高変化率（5日 vs 20日平均） | 注目が集まり始めてるか |
| テクニカル | RSI, MACD, ボリンジャーバンド | トレンド転換シグナル |
| バリュエーション | PER, PBR, 配当利回り | 割安度 |
| バリュエーション変化 | PER/PBRの変化率 | 割安に動いてるか |
| ファンダメンタル | 決算サプライズ（実績 vs コンセンサス） | 予想を上回った銘柄 |
| セクター | セクター内相対強度 | 業界の中で抜き出てきてるか |
| 信用 | 信用買い残の変化 | 個人投資家の期待度 |
| マクロ | 日経平均/TOPIX相対強度 | 市場全体との乖離 |

### データソースの候補

| ソース | 内容 | コスト |
|--------|------|--------|
| **yfinance** | Yahoo Finance日本株データ（株価・出来高） | 無料 |
| **J-Quants API** | 東証公式データ（ファンダメンタル含む） | 無料プランあり |
| **kabutan** | 決算・財務データ | スクレイピング |
| **stooq** | ヒストリカルデータ | 無料 |

### program.md に書くルール

```
## 目標
日本株（東証プライム・スタンダード）の中から、
1〜3ヶ月後に値上がりする確率が高い銘柄を見つけるスコアリングモデルを構築する。

## 評価指標
- 主指標: スコア上位20銘柄の40営業日後平均リターン
- 副指標: TOPIX対比の超過リターン、Sharpe Ratio
- ベースラインを上回らない変更は破棄する

## 禁止事項
- 未来データのリーク（テスト期間のデータを訓練に使わない）
- テスト期間での過学習（テスト期間のハイパーパラメータチューニング禁止）
- prepare.py の変更

## 訓練/テスト期間
- 訓練: 2015-01-01 〜 2023-12-31
- テスト: 2024-01-01 〜 2025-12-31

## 探索の方向性
- まずLightGBMで特徴量の有効性を検証
- 特徴量の組み合わせ・変換を重点的に探索
- モメンタム + バリュエーション の組み合わせが有望
- セクター中立化も試す
```
